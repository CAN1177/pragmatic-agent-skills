# Contributing

Contributions are welcome if they make the repository more reusable and more rigorous.

## Good Contributions

- improve a skill's trigger conditions or non-goals
- tighten fallback logic
- add realistic examples
- add focused validation or tests
- improve setup or documentation clarity

## Avoid

- broadening a skill until it loses focus
- adding generic prompt collections without a clear execution contract
- mixing personal data with public templates or shared assets

## Preferred Shape Of A Skill

A strong skill usually includes:

- a clear one-line purpose
- trigger conditions
- non-goals
- explicit workflow or fallback logic
- examples
- optional scripts/templates only when they materially help
